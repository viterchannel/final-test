Required Changes
1. Backend – Add Bypass Check to Auth Endpoints
Modify POST /auth/send-otp and POST /auth/verify-otp:

Create a reusable helper checkOtpBypass(phone: string) that returns:

ts
{ active: boolean, expiresAt?: Date, reason?: 'global' | 'user' | 'whitelist' }
Logic:

Global suspension from platform_settings (key otp_disabled_until)

Per‑user bypass from otp_bypass table (check user_id via phone lookup)

Whitelist from otp_whitelist (check if phone matches any entry with enabled=true)

/auth/send-otp :

Call checkOtpBypass(phone)

If active === true → DO NOT send SMS. Return:

json
{ "bypass": true, "expiresAt": "...", "message": "OTP bypass active, no SMS sent" }
Else → proceed with normal OTP generation & SMS.

/auth/verify-otp :

Call checkOtpBypass(phone)

If active === true → skip OTP code validation, generate auth tokens and return success immediately.

Else → validate OTP code as usual.

Add a new endpoint (or reuse config) for apps to fetch bypass status without triggering SMS:
GET /auth/otp-status?phone=... → returns { bypassActive, bypassExpiresAt }

artifacts/ajkmart/hooks/useOTPBypass.ts
